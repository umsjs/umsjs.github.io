public

Defined in ums.js:1072

接受或拒绝一个刚建立的连到该RTMPServer的TCP连接.如果要拒绝一个刚建立 的连到该RTMPServer的TCP连接，isAccept应为true;反之，如果要拒绝一个TCP连接， isAccept应为false。

Parameters:

  • sockInfo Object

    新建立的TCP连接信息，该参数必须由'connection' 事件的监听回调函数传入

  • isAccept Boolean

    是否接受该TCP连接

Example:

server.accept(sockInfo, isAccept);

public async

Defined in ums.js:977

为特定事件添加一个监听处理程序. 当前支持'connection/reject/error/close'事件。

Parameters:

  • event_name String

    要订阅的事件名称

  • callback Function

    事件处理函数，事件处理函数的第一个参数是 新建立的TCP连接的基本信息，包括本端IP地址 和对端IP地址

Example:

server.on('connection', function(sockInfo){
    console.dir(sockInfo);
});

RTMPServer关闭事件及其事件监听处理程序. 如果添加'close'事件监听函数，则当RTMPServer关闭时回调函数将被调用。

connection

Defined in ums.js:994

新连接事件及其事件监听处理程序. 当一个新的TCP连接建立时，'connection'事件将被抛出，如果事先注册 有该事件的监听处理函数，则会调用该处理函数；否则，该TCP连接将被 默认接受。

Event Payload:

  • sockInfo Object

    新建立的TCP连接的IP信息

Example:

server.on('connection', function(sockInfo){
    console.dir(sockInfo);
});

错误事件及其事件监听处理程序. 如果添加'error'事件监听函数，则当出现内部错误时回调函数将被调用。
