public async

Defined in ums.js:3381

向远端RTSP服务器发起一个连接建立操作。

Parameters:

  • url String

    要连接的RTSP服务器的地址，地址形式： rtsp://ip:port/stream_name

  • properties Object

    连接参数集合，需含有发布到本地的流名称参数，形如： { streamName : 'livestream' }

  • callback Function

    异步调用完成回调函数

Example:

var props = { streamName : 'livestream' };
conn.connect('rtsp://127.0.0.1:554/stream', props, function(event){
    console.dir(event);
    switch(event.code){
    case 'RTSPConnection.Connect.Success':
        // Do continue work
        break;
    case 'RTSPConnection.Connect.Failed':
        conn.close();
        break;
    }
});

为特定事件添加一个监听处理程序. 当前支持除'_error/_result/onStatus/error/close'等事件之外的 任何自定义事件.上述事件为内部保留事件，用户不应定义与这些事件同名的 远程调用函数名。与用户自定义事件相对应的事件处理函数的参数列表与远端 调用时传入的参数列表一致。

Parameters:

  • event_name String

    要订阅的事件名称

  • callback Function

    事件处理函数

Example:

connection.on('error', function(err){
    if (err) {
        console.dir(err);
    }
});

openNotify

  • eNotifyType
  • callback
)
public

Defined in ums.js:3358

设置开启/关闭命令发送完成通知(该接口主要用于开发调试，建议生产环境 中关闭命令发送完成通知功能以提高运行效率). 如果设置命令发送完成通知功能，则每一个RTMP RPC命令在成功发送完成 后，都可以通过监听'cmdSentDone'事件来捕获发出去的命令包的拷贝。 异步设置完成回调函数的第一个参数为出错信息，如果出错信息为null， 则表明设置成功，否则，表明设置失败。

Parameters:

  • eNotifyType Enum of mask

    是否开启相应的事件通知,可以是下列 值之一,或是其中几项的组合(通过|组合):
        ums.CONST.NOTIFY_ACTIVECHECK

  • callback Function

    异步设置完成回调函数

Example:

connection.openNotify(ums.CONST.NOTIFY_ACTIVECHECK,
    function(err){
        if (err) {
            console.dir(err);
        }
});
